# Zerox 分类与收入基础改造需求和流程

## 目标

Zerox 对原 Zero 的基础记账结构做了三类改造：

- 支出账单删除备注语义，只保留标题作为账单说明。
- 支出分类升级为二级分类，显示为 `大类·小类`。
- 新增收入记录、收入分类和收入统计。

原底部“债务”入口已调整为“收入”，债务功能保留在收入页的第二个分段中。

## 数据模型

### categories

- `name`
- `kind`: `expense` 或 `income`
- `parent_id`: 支出小类指向支出大类；支出大类和收入分类为空
- `category_status`
- `user_id`
- `icon`
- `color`

规则：

- 支出大类：`kind = expense`，`parent_id = null`
- 支出小类：`kind = expense`，`parent_id = 大类 id`
- 收入分类：`kind = income`，`parent_id = null`
- 只有大类独立管理颜色；小类展示时跟随大类颜色。

### expenses

- `title`
- `amount`
- `category_id`: 指向支出小类
- `user_id`
- `date`

规则：

- 不再展示或编辑备注。
- 旧备注字段只作为兼容遗留数据存在，不作为用户功能。

### incomes

- `title`
- `amount`
- `category_id`: 指向收入分类
- `user_id`
- `date`

规则：

- 收入与支出分表存储。
- 收入分类保持一级结构。

## 默认分类

默认支出分类来自 `docs/支出与收入分类明细.xlsx`，并固化到 `src/constants/defaultCategories.ts`。

默认收入分类：

- 工资
- 收入
- 投资收入
- 兼职
- 生活费
- 红包
- 二手闲置
- 借入
- 报销
- 退税
- 理财

语言规则：

- 新用户默认分类跟随引导页语言生成中文或英文。
- 老用户已有分类不自动翻译。
- 用户自定义分类会保留原文。

## 老数据简单映射

旧分类按固定规则映射：

- 餐饮 -> 餐饮·午餐
- 交通 -> 出行·交通
- 娱乐 -> 娱乐·电影
- 医疗 -> 医疗·医院
- 购物 -> 购物·生活用品
- 旅行 -> 娱乐·旅游
- 教育 -> 教育·教育
- 礼物 -> 人情·礼物
- 其他或未匹配 -> 其他·其他

不做复杂迁移向导。未匹配内容交给用户后续自行编辑。

## UI 规则

### 支出分类选择

- 默认展示所有大类，不展开任何一个大类。
- 点击大类后展开该大类的小类。
- 切换到其他大类时，收起之前的大类。
- 展开后小类只显示小类名，不重复显示 `大类·小类`。

### 分类管理

- 支出 / 收入使用分段切换。
- 支出页默认收起所有大类。
- 大类右侧保留展开 / 收起按钮。
- 展开后显示小类，小类只显示自身名称。
- 支出大类和收入分类使用左滑删除。
- 删除采用和账单一致的撤销逻辑，撤销时间结束后再真正删除。

### 首页流水

- 首页同时展示支出和收入的单条流水。
- 支出和收入使用不同金额颜色区分。
- 不在首页额外增加收入汇总模块，保持原来简洁风格。

### 统计页

- 统计页按当前用户的大类聚合支出。
- 点击大类后展示该大类下的小类饼图和消费明细。
- 小类统计进入时会为小类分配可区分颜色，避免同色饼图不可读。
- 删除统计明细中的账单后，数据库和页面状态立即同步，不保留幽灵账单。

### 收入页

- 顶部为收入 / 债务分段。
- 收入分段展示月收入、收入条数、分类分布、收入流水和新增收入入口。
- 债务分段保留原债务管理能力。

## 导入导出

- 导出包含用户、分类、支出、收入、货币、债务等数据。
- 分类导出包含 `kind` 和父级信息。
- 支出导出不再面向用户暴露备注。
- 收入支持导入导出。
- 老版本导入时按简单映射规则处理旧分类，未匹配进入 `其他·其他`。
- API Key 不进入导入导出文件。

## AI 兼容

- AI 创建记录时支持支出和收入。
- AI 分类候选来自当前用户真实分类，因此用户自定义大类 / 小类可以被 LLM 看到。
- LLM 不直接决定 `categoryId`，本地代码负责匹配。
- 支出优先匹配小类，收入匹配一级收入分类。

## 固定开发流程

1. 先写需求文档和默认分类常量。
2. 再升级 WatermelonDB schema、model、service。
3. 再调整 Redux 数据流。
4. 再改支出、分类、统计 UI。
5. 再新增收入页面和收入录入能力。
6. 再把债务嵌入收入页面第二页。
7. 再升级导入导出和 AI 兼容。
8. 再做性能审查和缓存失效检查。
9. 不经用户确认，不运行 lint、test。
10. 只在用户明确要求后运行 Android / iOS 构建。

## 验收标准

- 新增 / 编辑支出不再出现备注。
- 支出分类以大类展开、小类选择的方式展示。
- 首页账单列表同时显示支出和收入。
- 统计页能按大类聚合，并可进入小类明细。
- 分类页能管理支出大类 / 小类和收入分类。
- 删除大类、收入分类和账单时均有撤销时间。
- 收入页能新增收入、展示月收入、收入条数、分类分布和收入流水。
- 债务功能仍可使用。
- AI 快速记账能创建支出和收入。
- 导入导出在新结构下可用。
