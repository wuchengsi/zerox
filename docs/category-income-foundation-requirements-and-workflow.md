# Zero 分类与收入基础改造需求和流程

## 目标

本阶段改造 App 基础记账结构，不扩展 LLM 能力：

- 支出账单删除备注语义，只保留标题作为账单说明。
- 支出分类升级为二级分类，显示为 `大类·小类`。
- 新增收入记录、收入分类和收入统计。
- 原底部“债务”入口改为“收入”，债务保留在收入页第二页。

## 数据模型

- `categories`
  - `name`
  - `kind`: `expense` 或 `income`
  - `parent_id`: 支出小类指向支出大类；支出大类和收入分类为空
  - `category_status`
  - `user_id`
  - `icon`
  - `color`
- `expenses`
  - `title`
  - `amount`
  - `category_id`: 指向支出小类
  - `user_id`
  - `date`
- `incomes`
  - `title`
  - `amount`
  - `category_id`: 指向收入分类
  - `user_id`
  - `date`

## 默认分类

支出默认分类来自 `docs/支出与收入分类明细.xlsx`，并固化到代码常量中。收入默认分类为：

- 工资
- 收入
- 投资收入
- 兼职
- 生活费
- 红包
- 二手闲置
- 借入
- 报销
- 退税
- 理财

## 老数据简单映射

老分类按固定规则映射：

- 餐饮 -> 餐饮·午餐
- 交通 -> 出行·交通
- 娱乐 -> 娱乐·电影
- 医疗 -> 医疗·医院
- 购物 -> 购物·生活用品
- 旅行 -> 娱乐·旅游
- 教育 -> 教育·教育
- 礼物 -> 人情·礼物
- 其他或未匹配 -> 其他·其他

不做复杂迁移向导。未匹配内容交给用户后续自行编辑。

## 固定开发流程

1. 先写需求文档和默认分类常量。
2. 再升级 WatermelonDB schema、model、service。
3. 再调整 Redux 数据流。
4. 再改支出、分类、统计 UI。
5. 再新增收入页面和收入录入能力。
6. 再把债务嵌入收入页面第二页。
7. 再升级导入导出和 AI 最小兼容。
8. 不经用户确认，不运行 `npm run typecheck`、lint、test。
9. 不经用户明确要求，不运行 Android/iOS 构建。

## 验收标准

- 新增/编辑支出不再出现备注。
- 支出分类以 `大类·小类` 展示和选择。
- 首页账单列表显示二级分类。
- 统计页能按大类聚合，并可进入明细。
- 分类页能管理支出大类/小类和收入分类。
- 收入页能新增收入、展示月收入、收入条数、分类分布和收入流水。
- 债务功能仍可使用。
- AI 快速记账仍可创建支出。
- 老分类能简单映射；未匹配进入 `其他·其他`。
